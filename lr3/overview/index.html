REST API для управления медицинским приложением

Реализованы CRUD-операции

  • Реализованы для всех ключевых моделей:
  • Patient
  • MedicalRecord
  • Diagnosis
  • Service
  • Doctor
  • ConsultingRoom
  • Visit
  • DoctorSchedule

  • Payment

  • Эндпоинты поддерживают:

  • Получение списка записей.
  • Получение конкретной записи по ID.
  • Создание, обновление и удаление записей.

Кастомные отчёты

  1. Отчёт 1: Список пациентов конкретного врача с датами и стоимостью их визитов.
  2. Отчёт 2: Телефоны пациентов, которые посещали отоларинголога и родились после 1987 года.
  3. Отчёт 3: Список врачей, работающих в указанный день.
  4. Отчёт 4: Количество визитов по датам.
  5. Отчёт 5: Суммарная стоимость лечения пациентов, сгруппированная по дням и врачам.
  6. Отчёт 6: Список пациентов, которые уже оплатили лечение.
  7. Отчёт 7: Подробный отчёт о работе врачей за указанный период, включая общий доход и информацию о визитах.

Использование связей между моделями

  • Предварительная загрузка связанных данных (services, visits и т.д.) через prefetch_related и select_related улучшает производительность.
  • Агрегации (annotate, aggregate) используются для подсчёта стоимости визитов и других аналитических данных.

Валидация и права доступа

  • Применяется permissions.IsAuthenticated, что обеспечивает безопасность API, требуя аутентификации пользователя.
  • Валидация данных через сериализаторы предотвращает запись некорректной информации.

Обработка ошибок

  • Корректная обработка ошибок, например:
  • Возврат 404 для несуществующих записей.
  • Возврат 400 для некорректных запросов.
