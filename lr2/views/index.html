Views

Этот файл содержит представления для управления функциональностью "Доски Заданий", включая регистрацию, вход, просмотр и сдачу заданий, а также административные действия.

Основные представления

base_view

Начальная точка для отображения основного шаблона.

Параметры - request: HTTP-запрос.

Шаблон - base.html

register_student

Представление для регистрации нового студента.

Параметры - request: HTTP-запрос.

Логика - Если запрос POST, проверяет валидность формы регистрации StudentRegistrationForm и сохраняет нового студента. - При успешной регистрации перенаправляет на страницу входа.

Шаблон - account/register.html

login_student

Представление для входа студента в систему.

Параметры - request: HTTP-запрос.

Логика - Если запрос POST, проверяет введенные данные с помощью StudentLoginForm. - При успешной аутентификации сохраняет идентификатор пользователя в сессии. - В случае ошибки выводит соответствующее сообщение.

Шаблон - account/login.html

logout_student

Представление для выхода студента из системы.

Параметры - request: HTTP-запрос.

Логика - Удаляет идентификатор пользователя из сессии и перенаправляет на страницу входа.

Проверка аутентификации

student_authenticated

Функция для проверки, аутентифицирован ли текущий пользователь (основана на его user_id).

Параметры - user_id: Идентификатор пользователя, хранящийся в сессии.

Возвращает - True, если пользователь найден в базе данных, иначе False.

Представления заданий

task_list

Представление для отображения списка всех заданий.

Параметры - request: HTTP-запрос.

Логика - Проверяет аутентификацию студента через student_authenticated. - Отображает список всех заданий и оценки студента по каждому из них (если имеются).

Шаблон - tasks/task_list.html

submit_assignment

Представление для отправки выполненного задания студентом.

Параметры - request: HTTP-запрос. - task_id: Идентификатор задания для отправки.

Логика - Проверяет, был ли студент аутентифицирован. - Проверяет, было ли задание уже отправлено, если да — выводит сообщение. - Если задание еще не отправлено, создает новое или обновляет существующее, устанавливая статус submitted.

Шаблон - submit_assignment.html

edit_assignment

Представление для редактирования задания (домашнего задания).

Параметры - request: HTTP-запрос. - assignment_id: Идентификатор задания для редактирования.

Логика - Проверяет, аутентифицирован ли студент. - Загружает задание и предоставляет форму для редактирования ответа на него. - Сохраняет изменения, если форма валидна.

Шаблон - assignments/edit_assignment.html

Административные представления

create_task

Представление для создания нового задания (доступно только для администратора).

Параметры - request: HTTP-запрос.

Логика - Проверяет, является ли пользователь суперпользователем. - Если запрос POST, проверяет валидность формы TaskCreationForm и сохраняет новое задание. - Перенаправляет на список заданий при успешном создании.

Шаблон - tasks/create_task.html

admin_grades_list

Представление для отображения всех оценок студентов (доступно только для администратора).

Параметры - request: HTTP-запрос.

Логика - Загружает все задания с оценками студентов, чтобы администратор мог просматривать успеваемость студентов.

Шаблон - grades/admin_grades.html

Представление для отображения оценок студента

show_only_my_grades

Представление для отображения только оцененных заданий конкретного студента.

Параметры - request: HTTP-запрос.

Логика - Проверяет аутентификацию студента. - Загружает все оцененные задания студента и передает их в шаблон для отображения.

Шаблон - grades/show_only_my_grades.html
