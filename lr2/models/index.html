Модели

Модели описывают структуру базы данных и основные сущности приложения "Доска Заданий". Здесь приведены модели для представления классов, студентов, заданий и выполненных заданий.

StudentClass

StudentClass представляет учебный класс, к которому может быть привязан студент.

Поля

  • class_name: CharField
  • Максимальная длина: 2 символа
  • Уникальное название учебного класса (например, "10A").

Методы

  • str: возвращает название учебного класса.

Student

Student представляет зарегистрированного студента и связан с моделью StudentClass через связь "многие к одному".

Поля

  • username: CharField
  • Максимальная длина: 150 символов
  • Уникальное имя пользователя студента.
  • name: CharField
  • Максимальная длина: 100 символов
  • Имя студента.
  • email: EmailField
  • Уникальный email адрес студента.
  • password: CharField
  • Максимальная длина: 128 символов
  • Хранит зашифрованный пароль студента.
  • assigned_class: ForeignKey
  • Ссылка на модель StudentClass, которая удаляется как SET_NULL (если класс удален, ссылка станет NULL).
  • related_name="students" — позволяет обратный доступ к списку студентов класса.

Методы

  • str: возвращает username студента.
  • set_password(raw_password): сохраняет зашифрованный пароль студента с использованием библиотеки bcrypt.
  • check_password(raw_password): проверяет соответствие введенного пароля с сохраненным паролем с использованием bcrypt.

Task

Task представляет учебное задание, которое может быть назначено нескольким классам студентов.

Поля

  • title: CharField
  • Максимальная длина: 120 символов
  • Название задания.
  • description: TextField
  • Текстовое описание задания.
  • date_issued: DateTimeField
  • Дата создания задания. По умолчанию используется текущее время (timezone.now).
  • deadline: DateField
  • Дата завершения задания.
  • created_by: CharField
  • Имя создателя задания.
  • applicable_classes: ManyToManyField
  • Ссылка на StudentClass, указывающая, какие классы могут выполнить это задание.
  • related_name="tasks" — позволяет получить задания, связанные с конкретным классом.

Методы

  • str: возвращает название задания и срок сдачи (deadline).
  • overdue: свойство, проверяющее, просрочено ли задание. Возвращает True, если текущая дата позднее deadline.

Assignment

Assignment представляет выполненное задание, связанное с определенным студентом и заданием.

Поля

  • task: ForeignKey
  • Ссылка на модель Task. При удалении задания, связанные Assignment также удаляются.
  • related_name="assignments" — позволяет получить все выполненные задания для конкретного задания.
  • student: ForeignKey
  • Ссылка на модель Student. При удалении студента, связанные Assignment также удаляются.
  • related_name="assignments" — позволяет получить все выполненные задания для конкретного студента.
  • submission_text: TextField
  • Текст ответа на задание, который студент отправил.
  • Может быть пустым.
  • grade: PositiveSmallIntegerField
  • Оценка за задание (целое положительное число).
  • Может быть пустым (null).
  • submission_date: DateField
  • Дата отправки задания.
  • Может быть пустым (null).
  • reviewed_date: DateField
  • Дата проверки задания.
  • Может быть пустым (null).
  • status: CharField
  • Статус выполнения задания.
  • Значения: assigned, submitted, reviewed.
  • Значение по умолчанию: assigned.

Методы

  • str: возвращает строку, содержащую название задания, имя студента и текущий статус.

Класс Meta

  • unique_together: (task, student) — определяет, что одна и та же пара студент-задание должна быть уникальной.

Пример использования статусов

  1. assigned — Задание назначено студенту, но не выполнено.
  2. submitted — Студент отправил задание.
  3. reviewed — Преподаватель проверил задание и поставил оценку.
