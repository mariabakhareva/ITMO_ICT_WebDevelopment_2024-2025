Форма входа и работа с токеном

Основная функциональность

Компонент LoginForm

Этот компонент представляет собой форму для входа пользователя. Он позволяет пользователю ввести имя пользователя и пароль, а затем отправить эти данные на сервер для получения токена аутентификации. Если вход успешен, токен сохраняется в localStorage, и происходит перенаправление пользователя на главную страницу.

  1. Состояние формы:
  2. Компонент использует хук useState для хранения данных формы:
    • username: имя пользователя.
    • password: пароль.

  3. Также используется состояние error для отображения сообщений об ошибках.

  4. Обработка изменений в форме:

  5. Каждое изменение в полях формы (имя пользователя и пароль) обрабатывается функцией handleChange, которая обновляет состояние formData.

  6. Отправка данных на сервер:

  7. При отправке формы вызывается функция handleSubmit, которая выполняет POST-запрос на сервер для получения токена аутентификации. Адрес для запроса: http://127.0.0.1:8000/api/auth/token/token/login/.
  8. В случае успешного ответа от сервера, токен сохраняется в localStorage с помощью сервиса TokenStore. Также вызывается функция onLogin, уведомляющая родительский компонент о успешном входе, и происходит перенаправление на страницу /home.

Работа с токеном

Для аутентификации и взаимодействия с сервером используется токен, который хранится в localStorage и добавляется в заголовки запросов с помощью интерцептора.

  1. Сервис TokenStore:
  2. Сервис TokenStore предоставляет методы для работы с токеном в localStorage:
    • getToken: извлекает токен из localStorage.
    • setToken: сохраняет токен в localStorage.
    • clearToken: удаляет токен из localStorage.

Токен хранится под ключом authToken.

  1. Интерцептор Axios:
  2. Для автоматического добавления токена в заголовки всех исходящих запросов используется интерцептор axiosInstance. Этот интерцептор перехватывает все запросы, проверяет наличие токена в localStorage, и если токен найден, добавляет его в заголовки запросов в формате Authorization: Token <token>.

  3. Это позволяет обеспечить аутентификацию при каждом запросе, не требуя вручную добавлять токен в каждый запрос.

  4. Axios Instance:

  5. Конфигурация для использования axios с предустановленным базовым URL (http://127.0.0.1:8000/api/v1/) и интерцептором для авторизации запросов.
