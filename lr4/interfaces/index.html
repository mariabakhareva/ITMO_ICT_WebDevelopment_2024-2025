Интерфейс списка врачей

Компонент DoctorsList отображает список врачей и предоставляет функциональность для добавления, редактирования и удаления информации о врачах.

Основная функциональность

  1. Отображение списка врачей:
  2. Загружает данные о врачах с сервера через axiosInstance и отображает их в виде таблицы.

  3. Для каждого врача отображаются: имя, фамилия, специальность, образование, пол и дата рождения.

  4. Поиск:

  5. Включает поле для поиска врачей по имени или фамилии. Поиск выполняется по введенному запросу, фильтруя список врачей в реальном времени.

  6. Добавление и редактирование:

  7. Позволяет добавлять нового врача или редактировать существующего через модальное окно.
  8. Модальное окно содержит форму с полями для ввода данных (имя, фамилия, специальность и т.д.).

  9. При сохранении данных врач сохраняется через API. Если это редактирование, отправляется PUT-запрос, иначе POST-запрос для нового врача.

  10. Удаление врача:

  11. Предоставляет возможность удалить врача из списка, вызвав соответствующий запрос на сервер.

  12. Иконки действий:

  13. Для каждого врача в таблице предусмотрены иконки для редактирования (Edit), удаления (Delete) и отображения расписания (Schedule).

  14. Модальное окно:

  15. Модальное окно для добавления или редактирования врача содержит все необходимые текстовые поля. При отмене или успешном сохранении форма закрывается.

Интерфейс списка пациентов

Компонент PatientsList управляет отображением списка пациентов, а также предоставляет функциональность для поиска, добавления, редактирования и удаления информации о пациентах.

Основная функциональность

  1. Отображение списка пациентов:
  2. Загрузка и отображение списка пациентов с сервера через API-запрос с использованием axiosInstance.

  3. Для каждого пациента отображаются следующие данные: имя, фамилия, дата рождения, телефон и email.

  4. Поиск:

  5. Включает поле для поиска по имени или фамилии пациента. Поиск осуществляется в реальном времени по введенному запросу.

  6. Добавление и редактирование:

  7. Позволяет добавлять нового пациента или редактировать существующего через модальное окно.
  8. Модальное окно содержит форму с полями для ввода данных (имя, фамилия, дата рождения и т.д.).

  9. При сохранении данных пациента выполняется запрос POST (для добавления) или PUT (для редактирования).

  10. Удаление пациента:

  11. Возможность удаления пациента из списка, с последующим обновлением данных через API.

  12. Иконки действий:

  13. Для каждого пациента в таблице предусмотрены иконки для редактирования (Edit), удаления (Delete).

  14. Модальное окно:

  15. Модальное окно для редактирования или добавления пациента включает форму с полями для ввода имени, фамилии, даты рождения, телефона и email.
  16. При отмене или успешном сохранении форма закрывается.

Список оплат

Компонент PaymentsList отображает таблицу с оплатами, с возможностью фильтрации по врачу, пациенту и дате.

Основные функции:

  • Фильтрация: позволяет фильтровать данные по врачу, пациенту и дате.
  • Загрузка данных: асинхронно загружает данные о врачах, пациентах и оплатах с сервера.
  • Таблица: отображает информацию о визитах, дате, сумме, методе оплаты и статусе успешности.
  • Индикатор загрузки: отображается во время загрузки данных.
  • Обработка состояния: при изменении фильтров автоматически обновляется список оплат.

Список кабинетов

Компонент ConsultingRoomsList позволяет управлять списком кабинетов с возможностью добавления, редактирования и удаления данных.

Основные функции:

  • Фильтрация: поиск по номеру кабинета.
  • Добавление/Редактирование: форма для ввода данных о кабинете и ответственном враче.
  • Удаление: возможность удаления кабинета.
  • Модальные окна: для добавления и редактирования кабинетов.

Поля формы:

  • Ответственный врач (ID)
  • Номер телефона
  • Дата работы
  • Время работы (с и до)

Список визитов

Компонент VisitsList предоставляет возможность управлять визитами пациентов в медицинском учреждении. Он включает функции добавления, редактирования, удаления визитов и поиска по ключевым данным.

Основные функции:

  • Поиск визитов: просмотр информации о визитах пациентов.
  • Добавление/Редактирование визита: форма для ввода и редактирования данных о визите, таких как пациент, врач, дата и время визита, диагнозы, рекомендации.
  • Удаление визита: возможность удаления визита из списка.

Структура формы:

  • ID пациента и врача
  • Дата и время визита
  • Текущее состояние пациента
  • Рекомендации по лечению
  • Диагнозы и услуги (через запятую)

Взаимодействие с сервером:

  • Axios для загрузки и сохранения данных визитов, пациентов, врачей, диагнозов и услуг.
